读的时候不记账，
写的时候准翻船

文献调研最大的浪费，是重读一遍只为找回那句话的出处。CiteDebt 把记账成本压到一次按键。

选中即捕获

网页里选中文字，按 ⌥⇧C 或右键「立一张出处欠条」。摘录进剪贴板，出处同步归档，阅读节奏零打断。

欠条自动记账

每条欠条记下选区、前后最多 160 字上下文、页面标题、URL、时间与项目。侧边栏按状态、项目、关键词随时筛选。

写完集中结算

勾选欠条，一键输出 Markdown / 纯文本 / HTML 来源清单，自动去重相同来源。复制进论文，结清销账。

PRIVACY BY ARCHITECTURE

隐私不是功能，是架构

CiteDebt 从第一天起按「数据不出浏览器」设计——这对处理未发表手稿与内部资料的科研场景尤其重要。

零网络传输不申请任何站点权限，内容安全策略禁止一切网络连接（connect-src 'none'），技术上就无法上传。
只存在本机欠条存于浏览器 IndexedDB，设置存于 chrome.storage.local。卸载或撤回同意，数据即彻底消失。
明确同意才工作首次使用必须阅读并同意隐私说明；未同意时不读取任何数据，也不响应捕获快捷键。
你的数据你做主随时导出全部 JSON；一键撤回同意并清空本机一切痕迹；敏感域名可列入排除清单。

PRIVACY POLICY

隐私政策

生效日期：2026 年 7 月 19 日 · 适用产品：CiteDebt 引文欠条（Chrome / Edge 扩展，v0.2.0）

我们处理哪些数据

CiteDebt 只在你主动触发捕获（快捷键或右键菜单）时处理以下数据：

  • 你主动选择的网页文字（纯文本摘录）；
  • 选区前后各最多 160 字符的上下文；
  • 页面标题与 HTTP(S) URL；
  • 捕获时间与你指定的项目名。

若选区疑似包含身份证号、银行卡号、手机号或邮箱等敏感信息，扩展会先弹出确认，由你决定是否登记。

数据存储在哪里

  • 欠条数据保存在本机浏览器的 IndexedDB 中；
  • 设置与隐私同意记录保存在 chrome.storage.local 中；
  • 所有数据均不离开你的设备。

数据会不会被传输

不会。CiteDebt 不申请任何站点权限（host permissions），扩展页面的内容安全策略包含 connect-src 'none'，从技术上禁止扩展页面发起任何网络请求。没有账号体系、云同步、遥测、分析、广告或远程代码。开发者与任何第三方都不会收到你的数据。

权限的使用方式

  • activeTab / scripting：仅在你主动触发时，向当前标签页注入一次性的捕获函数，读取选区与有限上下文；
  • clipboardWrite：把你已选择的纯文本写入剪贴板，写入失败时不登记欠条；
  • contextMenus：提供「立一张出处欠条」右键入口；
  • storage / sidePanel / alarms：保存本地设置与同意记录、展示欠条队列、在本地重算到期提醒角标。

导出、导入与删除

  • 你可以在设置页随时导出全部数据为 JSON 文件（由你主动发起，保存位置由你选择）；
  • 导入仅接受你自己选择的 JSON 文件；
  • 在设置页点击「撤回同意并清除数据」，经确认后，全部欠条、设置与同意记录将从本机彻底删除；
  • 你可以在设置页将敏感域名加入排除清单，扩展在这些页面上不会登记欠条。

儿童隐私

CiteDebt 不面向 13 岁以下儿童，也不会有意识地收集儿童数据。

政策变更

本政策如有变更，将更新本页面的生效日期。由于扩展不进行任何网络通信，变更不会追溯影响你已存储在本机的数据。
